0.5.0
* 機能追加
・メソッドにパラメータとして扱わない「noparam」メソッドパラメータを追加しました。
noparamに設定されているパラメータはDaoメソッド内で用いるものとなり、SQL文には反映されません。
・SQLファイル、SQLメソッドパラメータ以外でnullを明示的に指定できる「nullParam」メソッドパラメータを追加しました。
nullParamに設定されているパラメータは条件式や設定値に明示的にnullになります。

0.4.0
* 仕様変更
・Daoの作成方法が変わります。
0.3.1以前とコード互換がありません。
Daoオブジェクト作成処理を置き換る必要となります。

（0.3.1以前）
$dao = new KitazDao("Table1Dao", "customConfig.config");

（0.4.0以降）
$kdao = new KitazDao("customConfig.config");
$dao = $kdao->getDao("Table1Dao");

「new KitazDao()」により、共有できるPDOオブジェクトを作成します。
PDOオブジェクトのクラスにgetDaoメソッドを実行し、
過去のKitazDaoのようにクラス名を指定します。
KitazDaoオブジェクト作成時に任意の設定ファイルを記述すると、
その設定ファイルを用いることになりますが、
省略すると従来通り「KitazDao/KitazDao.config」を参照します。

このバージョンより、トランザクション処理をKitazDaoクラスに移動しており、
getDaoにはトランザクションのメソッドが存在しなくなりました。

* 機能追加
・コードアシスト機能を利用してコーディングしやすくすることも可能になります。
getDaoメソッドに第２パラメータを渡すことで参照渡しによりKitazDaoのDaoを渡します。

$kdao = new KitazDao();
$dao = new Table1Dao();
$kdao->getDao("Table1Dao", $dao);

事前に$dao変数に作成したDaoクラスを宣言することにより、
開発環境上でのコードアシスト機能で、作成したメソッドが候補に出てきます。
getDaoメソッドにより$dao変数がKitazDaoが作成するクラスに置き換わりますが、
コードアシスト機能を維持させたままにできます（Eclipse4.2にて検証）。

・名称変更
ユーティリティの呼称を変更します。
変更前：util/egen.php→変更後：utils/egen.php

* 既知の問題(Oracle11g)
SELECT文でCLOBやBLOBを開いた状態でcommit()するとfatal errorを吐く。
